# Episode Development Script Template

> **定位**：编剧室开发版分集模板。含本集定位、场次资产声明、**叙事体幕级正文**、衔接、本集复盘等，用于 STAGE 7 **默认**拆解。交付版见 `agent/templates/Episode Final Script Template.md`（与本文不得混用于同一轮输出）。
>
> **单次使用范围（硬规则）**：**每一次** Agent 回复**只准输出「一集」**（仅一个 `## 第[集数]集`）。下一集须在主创确认后的下一轮再输出。
>
> **集-场-幕（硬规则）**：
>
> - **幕** = 下游 AI 视频可生成的**最小叙事单元**，单幕按 **≤15 秒** 设计；剪辑后时长会缩水，须**多幕**补足。
> - 单集约 **2 分钟**量级时，**全集幕数合计须 ≥ 8**（建议 **8～12 幕**）；本场次仅 **1～3 场**时，须通过**增加每场次幕数**达标，禁止用粗粒度「一整场一段」糊弄。
> - Markdown 层级固定：`**## 第N集` → `### 场次k` → `#### 幕m`**；**不得**把幕写成 `###`（会与场次同级混淆）。
>
> **资产标记体系（硬规则）**：
>
> - 场次头部**必须**用 ∆ 行声明本场出现的资产：`∆场景：@名称`、`∆角色：@名称 、 @名称`、`∆物品：@名称`。
> - **正文中每次**提到任何资产（角色、物品、场景）**必须**使用 `@名称` 格式，`@` 符号**绝对不能丢**。无论是叙事描写还是对白，只要提及设定集里的资产就必须加 `@`。
> - 所有 `@名称` **必须**与 STAGE 5 设定集中注册的名称**完全一致**，一字不差。
> - 若需引入新资产，须先回到 STAGE 5 设定集追加注册。
>
> **正文写作规范（硬规则）**：
>
> - 幕的「正文」是**一整段连续的叙事体剧本**：按**线性时间顺序**讲故事——场景描写、动作、氛围、对白**全部融合在同一段正文里**，像讲故事一样一气呵成。
> - **对白直接嵌在叙事中**，不单独拆成子字段。当角色说话时，用 `@角色名 [可选动作]："台词"` 格式，不同角色的台词断行，然后继续叙事。
> - **禁止**在正文内再拆出 `对白要点：`、`动作/画面描述：`、`画面/镜头：`、`动作：`、`潜台词：` 等子标签——幕下**只有三个字段**：`时长`、`正文`、`衔接`，没有第四个。
> - 这是**剧本**，不是分镜脚本——**不写镜头、景别、机位**，只讲故事。镜头设计留给下游制作阶段。
> - 每幕正文须**充实饱满**——15 秒的叙事量意味着需要写清动作细节、环境氛围、角色微表情和情绪变化，不可只写一两句就草草了事。
> - 对白必须去 AI 味、保留攻防、能短就短、能用动作解决的不用台词硬说。
>
> **正文写法示例**（仅格式示意）：
> ```
> 深夜的集团大厦死一般寂静。@角色A 抱着一摞加急文件，拖着灌铅般的双腿走进空无一人的电梯。他靠在冰冷的轿厢壁上，闭着布满红血丝的眼睛，酸痛地揉了揉后颈，按下楼层键。
> @角色A [轻声叹息]："打三份工的代价，就是连呼吸都觉得奢侈。"
> 电梯面板的灯光突然疯狂闪烁，发出刺耳的电子警报声。@角色A 猛地睁眼，下意识攥紧文件。
> ```
> 注意：描写→对白→描写**一气呵成**，没有任何子标签隔断。
>
> **示例≠成稿**：下文「场次 / 每场下的幕」仅为**排版示意**；以本集 **「本集幕数规划」** 为准，**可增删** `### 场次` 与 `#### 幕`，全集合计仍须 **≥8 幕**；**禁止**因示例里只有两场、每场四幕就只写两场四幕。
