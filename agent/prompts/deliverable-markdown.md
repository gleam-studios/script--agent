# 交付物输出格式（编剧室）

- **所有模板交付物、结构化小节、列表与强调**，一律使用 **Markdown** 编写（`#` / `##` 标题、`-` 列表、`**粗体`**、表格、围栏代码块等），便于右侧产物面板渲染与验收。
- **避免**只输出无标题、无列表的纯闲聊式长段；需要说明流程时可用简短过渡句，但**主体内容**仍须是可拆分的 Markdown 结构。
- 与既有 STAGE 模板、知识库中的章节标题约定保持一致；不确定时优先用 `##` 级标题分段。
- **小节标题必须单独成行**：`## 标题` 前须换行，勿与上一句英文/中文粘在同行（例如错误：`…句号。## 核心关系定义`）；`人物矩阵总览`、`**## 主角一` / `## 主角二`**、`**## 配角一` / `## 配角二` …**（须带标头，勿仅以姓名起段）、`关键配角：…` 等也须以 `##`  开头，便于自动落入右侧产物槽位与解析。

## 资产标记体系（STAGE 5/6/7）

- `**∆`（三角）** = 资产分类标记。仅三种：`∆人物`、`∆物品`、`∆场景`。在 STAGE 5 设定集中作为 `##` 级标题使用。
- `**@`（at）** = 具体资产名称标记。格式为 `@中文名(英文名)` 或 `@中文名`，紧跟分类后列出。
- STAGE 5 设定集：`## ∆人物` 下列 `@约翰(John)` + 人设/特征；`## ∆物品` / `## ∆场景` 同理。
- STAGE 6 分集大纲：∆ 声明行写 `∆人物：@名称1，@名称2 ∆场景：@名称 ∆物品：@名称`；**「开头钩子」「本集剧情」「结尾悬念」正文中每次提到任何资产也必须写 `@名称`**，裸写名称视为格式违规。
- STAGE 7 分集剧本：场次头部须用 ∆ 行声明本场资产（`∆场景：@名称`、`∆角色：@名称`、`∆物品：@名称`）；幕的正文为叙事体剧本，对白用 `@角色名："台词"` 格式，**正文中每次提到任何资产（角色/物品/场景）必须写 `@名称`**，`@` 符号绝对不能丢，名称须与设定集完全一致。
- 新资产须先在 STAGE 5 追加注册，方可在 STAGE 6/7 引用。
