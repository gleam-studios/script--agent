# 短剧分集规格（EPISODE_SPECS）

## 适用范围

- **总集数**：常见 **30～60 集**，以项目为准可变；写作前先定主线里程碑，再反推集数（见 `03_SERIES_BIBLE.md`、`skills/skill-series-architecture.md`）。
- **单集时长**：**1～2 分钟**（成片）；剧本侧以**对白字数 + 动作/场记密度**控制体量。

## 与模板的一致性（硬规则）

- 每一轮回复**只输出一集**，遵守 `agent/templates` 顶部说明：仅一个 `## 第[集数]集` 块。
- **开发版**默认：`agent/templates/Episode Development Script Template.md`。
- **交付版**：仅当主创明确索要交付/成品/精简/可下游版本时，使用 `agent/templates/Episode Final Script Template.md`；**禁止**同轮混用两模板。

## 集-场-幕（与主控协议一致）

与 `agent/prompts/main_prompt.md` §1 **[D]**、两套分集模板顶部硬规则一致：

- **层级**：`## 第N集` → `### 场次` → `#### 幕`；幕**不得**写成 `###`。
- **幕**：最小叙事单元，单幕按 **≤15 秒** 设计。正文为**叙事体剧本**——按线性时间讲故事（场景/动作/对白/氛围），**不写镜头/景别/机位**，镜头设计留给下游制作。对白格式 `@角色名："台词"`，不同角色断行。
- **场次头部**：用 ∆ 行声明本场资产（∆场景/∆角色/∆物品），加「场次目标与结果」。
- **数量**：单集约 2 分钟成片量级时，**全集幕数合计须 ≥ 8**（建议 **8～12**）；场次仅 1～3 场时须**加密每场的幕数**，禁止「整场一段」糊弄。
- **规划字段**：开发版填 **本集幕数规划**；交付版填 **本集幕数**（见模板）。

## 体量假设（可改项目常量）


| 项      | 建议默认                    | 说明                      |
| ------ | ----------------------- | ----------------------- |
| 语速     | **4～5 字/秒**（中文对白，含简短气口） | 旁白/OS/留白多则下调有效密度        |
| 对白目标字数 | **约 150～350 字/集**       | 动作与沉默多可偏下限；信息密可偏上限但警惕超时 |
| 成片目标   | **60～120 秒**            | 以导演/平台剪辑为准              |


## 格式与呈现（按项目勾选）

- **竖屏**：是否在正文标注「竖屏构图提示」由项目约定；未约定则优先保证戏剧信息与对白节奏。
- **字幕/OS**：是否单独列出 OS 行；若写，需简短，避免挤占对白时长。
- **场次数**：1～2 分钟建议 **2～4 场**（非强制，类型极端时可调整并在本集定位中说明）。

## 校验

可用仓库根目录 `tools/episode-stats.mjs` 做字数、粗估时长与 **`#### 幕` 数量**检查（`--min-acts` / `--strict`）。
