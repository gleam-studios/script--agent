# Episode Final Script Template

> **最终交付版分集模板**：成品向正文，结构与开发版一致为 **集 → 场 → 幕**，便于下游解析与 AI 视频分段。**仅当**主创按主控协议明确索要交付版时使用；与 `Episode Development Script Template.md` **不得在同一轮输出中混用**。
>
> **单次使用范围（硬规则）**：**每一轮**回复**只准输出一集**（仅一个 `## 第[集数]集`）。下一集须在主创确认后的下一轮再输出。
>
> **幕（硬规则）**：单幕按 **≤15s** 最小叙事单元写作；**全集幕数合计须 ≥ 8**（建议 8～12）。层级固定：`### 场次k` → `#### 幕m`。
>
> **资产标记体系（硬规则）**：
>
> - 场次头部**必须**用 ∆ 行声明本场资产：`∆场景：@名称`、`∆角色：@名称 、 @名称`、`∆物品：@名称`。
> - **正文中每次**提到任何资产（角色、物品、场景）**必须**使用 `@名称` 格式，`@` 符号**绝对不能丢**。
> - 所有 `@名称` **必须**与 STAGE 5 设定集中注册的名称**完全一致**，一字不差。
> - 若需引入新资产，须先回到 STAGE 5 设定集追加注册。
>
> **正文写作规范（硬规则）**：
>
> - 幕的「正文」是**一整段连续叙事**：场景描写、动作、氛围、对白**全部融合**在同一段里，不拆子字段。
> - 对白直接嵌在叙事中：`@角色名 [可选动作]："台词"`，不同角色断行后继续叙事。
> - **禁止**在正文内拆出 `对白要点：`、`动作/画面描述：`、`画面/镜头：` 等子标签——幕下**只有** `时长` 和 `正文` 两个字段。
> - **不写镜头、景别、机位**——只讲故事，镜头设计留给下游。
> - 每幕正文须**充实饱满**，不可一两句草草了事。
>
> **示例≠成稿**：下文场次与幕块仅为**排版示意**；须按本集实际 **幕数规划** **增删** `### 场次` 与 `#### 幕`，全集合计仍 **≥8 幕**；不得机械照搬示例场次数与每场幕数。
