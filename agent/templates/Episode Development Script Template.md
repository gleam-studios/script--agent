# Episode Development Script Template

> **定位**：编剧室开发版 / 分析型分集模板。含本集定位、场次目标、**幕级可拍单元**、潜台词说明、本集复盘等，用于 STAGE 5 **默认**拆解。交付版见 `agent/templates/Episode Final Script Template.md`（与本文不得混用于同一轮输出）。
>
> **单次使用范围（硬规则）**：**每一次** Agent 回复**只准输出「一集」**（仅一个 `## 第[集数]集`）。下一集须在主创确认后的下一轮再输出。
>
> **集-场-幕（硬规则）**：
>
> - **幕** = 下游 AI 视频可生成的**最小叙事单元**，单幕建议按 **≤15 秒**（或 10–15 秒区间）设计；剪辑后时长会缩水，须**多幕**补足。
> - 单集约 **2 分钟**量级时，**全集幕数合计须 ≥ 8**（建议 **8～12 幕**）；本场次仅 **1～3 场**时，须通过**增加每场次幕数**达标，禁止用粗粒度「一整场一段」糊弄。
> - Markdown 层级固定：`**## 第N集` → `### 场次k` → `#### 幕m`**；**不得**把幕写成 `###`（会与场次同级混淆）。
>
> **示例≠成稿**：下文「场次 / 每场下的幕」仅为**排版示意**；以本集 **「本集幕数规划」** 为准，**可增删** `### 场次` 与 `#### 幕`，全集合计仍须 **≥8 幕**；**禁止**因示例里只有两场、每场四幕就只写两场四幕。
